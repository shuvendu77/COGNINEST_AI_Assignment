# COGNINEST_AI_Assignment

# Clinic NLP-to-SQL API

> Convert plain-English questions into SQL queries against a clinic management database — powered by **Vanna AI 2.0** and **Google Gemini 2.0 Flash**.

---

## What It Does

Send a question like *"Which doctor has the most appointments?"* and get back:

- The generated **SQL query**
- **Result rows** with column names
- An optional **Plotly chart** for time-series / categorical data
- A natural-language **answer** from the LLM

---

## Tech Stack

| Layer | Technology |
|---|---|
| LLM | Google Gemini 2.0 Flash (via Vanna AI 2.0) |
| API | FastAPI + Uvicorn |
| Database | SQLite (`clinic.db`) |
| Memory | Vanna `DemoAgentMemory` (in-process, Jaccard similarity) |

### `POST /chat`

```bash
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"question": "Which doctor has the most appointments?"}'
```

**Response:**
```json
{
  "message": "Doctor with ID 4 has the most appointments (68).",
  "sql_query": "SELECT doctor_id, COUNT(*) AS appointment_count FROM appointments GROUP BY doctor_id ORDER BY appointment_count DESC LIMIT 1",
  "columns": ["doctor_id", "appointment_count"],
  "rows": [[4, 68]],
  "row_count": 1,
  "chart": null,
  "chart_type": null
}
```

When the data is time-series or categorical the response also includes a `chart` object (Plotly figure JSON) and `chart_type: "plotly"`.

**Error codes:**

| Code | Meaning |
|---|---|
| `429` | Gemini rate limit hit — wait and retry |
| `500` | Unexpected server error — check server logs |

## Notes on Rate Limits

The Gemini **free tier** allows:
- ~15 requests per minute
- ~1,500 requests per day

The demo runner and test suite both add a **12-second gap** between questions automatically. If you hit the daily limit, either wait for midnight (US Pacific) or create a key from a different Google account at [aistudio.google.com](https://aistudio.google.com).

## 1. Project Description

This project exposes a REST API where a client sends a plain-English question (e.g. *"Which doctor has the most appointments?"*) and receives back:

- The generated **SQL query**
- The **result rows** and column names
- An optional **chart** (Plotly) when the data is time-series or categorical
- A natural-language **summary** from the LLM

The system uses **Vanna AI's agent framework** to orchestrate LLM calls, tool execution, and memory-augmented retrieval — no manual prompt engineering required.

**Database:** SQLite (`clinic.db`) with five tables:
`patients` · `doctors` · `appointments` · `treatments` · `invoices`

**Request flow:**

```
Client → POST /chat
  └─► chat controller
        └─► agent.send_message()
              ├─► Gemini 2.0 Flash  (LLM)
              ├─► SearchSavedCorrectToolUsesTool  (memory retrieval)
              ├─► RunSqlTool → ValidatedSqliteRunner  (SQL guard + execute)
              └─► VisualizeDataTool  (optional chart)
        └─► JSON response  { message, sql_query, columns, rows, chart }
```

## 5. Memory Seeding

The agent memory is seeded **automatically at server startup** with 15 curated question-and-SQL pairs. These help the LLM recall correct query patterns for common clinic questions without re-learning each time.

To seed (or inspect) the memory in isolation, without starting the full server:

```powershell
# From the Test/ directory
python -c "
import asyncio
from app.core.agent import agent_memory
from app.memory.seeder import seed_memory_instance
n = asyncio.run(seed_memory_instance(agent_memory))
print(f'Seeded {n} pairs')
"
```
